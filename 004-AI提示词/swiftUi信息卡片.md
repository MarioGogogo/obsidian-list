# 通用SwiftUI API信息卡片生成提示词

你需要生成一张**竖屏（9:16比例，适配手机竖屏显示）**的卡通风格信息图，主题为「SwiftUI编程之【目标SwiftUI API名称】（基于iOS15最新SwiftUI语法）」，具体要求如下，可直接替换【目标SwiftUI API名称】（如Alert、Text、Button等）使用：

## 一、整体风格（固定不变，适配所有SwiftUI API）

1.  风格：可爱卡通风，采用圆润线条、柔和清新配色（贴合iOS原生视觉风格），可搭配简约编程相关卡通元素（如卡通Swift小鸟、手机界面插画、代码文件图标、变量小图标等），整体视觉活泼易懂，适合编程学习者保存和查阅。

2.  字体搭配：所有非代码描述内容全部使用**手写风格中文字体**（模拟笔记书写效果，清晰易读，字号分层分明）；代码内容单独使用等宽编程字体，语法高亮显示，与手写内容区分明显。

3.  重点标注：统一使用「荧光笔（优先黄色、浅蓝色、粉色，不刺眼）」和「彩色笔（优先红色、绿色、蓝色，色彩鲜明）」对核心信息进行标注，标注样式自然不遮挡内容，标注对象包括：核心概念关键词、关键属性/修饰符、易错点、重要步骤、核心代码片段。

  

## 二、内容结构（通用适配所有SwiftUI API，非代码描述均为中文，内容丰富详实）

### 模块1：标题区

-  主标题：SwiftUI编程之【目标SwiftUI API名称】（手写字体，大号醒目，可搭配荧光笔描边或填充背景突出重点）

-  副标题：基于iOS15最新SwiftUI语法（手写字体，小号辅助，用彩色笔标注「iOS15」关键词）

-  点缀：搭配与【目标SwiftUI API名称】相关的卡通插画（如Alert对应卡通弹窗插画、Text对应卡通文字气泡插画），增加活泼感。

  

### 模块2：核心概述区

-  内容：用中文手写字体详细描述该API的核心定义、应用场景，明确其在iOS15 SwiftUI中的定位，表述如下模板可直接复用：「【目标SwiftUI API名称】是SwiftUI中用于【核心功能描述，如：展示弹窗提示、渲染文本内容、创建可点击按钮等】的核心组件/修饰符，在iOS15中，其语法更简洁高效，支持【iOS15专属特性，如：状态直接绑定、自定义样式简化、无需桥接UIKit等】，是构建iOS原生界面的必备工具之一」。

-  标注：用黄色荧光笔标注概述中的核心关键词（如API核心功能、「iOS15」、「状态绑定」等关键表述）。

  

### 模块3：iOS15 SwiftUI 【目标SwiftUI API名称】核心特性区

-  呈现形式：以有序列表形式呈现（手写字体，彩色笔标注序列号1、2、3...），每条特性表述简洁易懂，内容丰富不空洞。

-  内容要求：针对该API的iOS15专属特性、核心优势编写，每条特性需用荧光笔标注关键信息，模板参考：

  1.  【特性1】：如「语法简化」「状态驱动」「自定义性强」等，后续补充具体描述（如：无需依赖UIKit桥接，直接通过SwiftUI原生API构建，代码量减少50%以上）

  2.  【特性2】：针对API核心能力描述（如：支持多种样式自定义，满足不同界面设计需求）

  3.  【特性3】：针对使用体验/适配性描述（如：自动适配iOS15及以上系统屏幕，遵循iOS设计规范，兼容性优异）

  4.  【特性4】：补充专属优势（如：支持链式调用，与其他SwiftUI组件无缝衔接）

  

### 模块4：完整代码演示区（带明显代码框，通用适配所有API）

1.  代码框样式：浅灰色背景、白色圆角边框，与手写内容形成明显区分，代码框顶部用手写字体标注标题「iOS15 SwiftUI 【目标SwiftUI API名称】完整可运行代码」（粉色荧光笔标注标题中的API名称）。

2.  代码内容要求：基于iOS15最新SwiftUI语法编写完整可运行代码，包含必要的导入语句、视图结构体、状态变量（如需）、预览视图，同时添加中文注释说明代码功能，代码结构清晰。

3.  代码外部标注：在代码框外侧，用彩色笔（红、绿、蓝区分）搭配手写字体，对核心代码片段进行针对性注释说明，模板参考：

    -  红色彩色笔指向【核心状态变量/初始化代码】，标注「核心：【功能说明，如：控制弹窗显示隐藏的状态变量】」

    -  绿色彩色笔指向【API核心调用代码】，标注「iOS15原生语法：【功能说明，如：直接绑定状态调用弹窗】」

    -  蓝色彩色笔指向【API关键属性/样式配置】，标注「关键配置：【功能说明，如：弹窗标题/文本样式设置】」

  

### 模块5：核心知识点标注区

-  呈现形式：无序列表/要点式呈现（手写字体，每条用小图标点缀），搭配荧光笔标注重点内容。

-  内容要求：总结该API的核心使用知识点、关键语法规则，模板参考：

  1.  【知识点1】：如「状态绑定需使用$符号（黄色荧光笔标注$），绑定对应类型变量」

  2.  【知识点2】：如「【关键属性/修饰符】是必选配置，用于【功能说明】」

  3.  【知识点3】：如「支持【自定义选项】，可根据业务需求灵活配置」

  4.  【知识点4】：如「核心语法不可省略，否则会导致编译失败/功能异常」

  

### 模块6：注意事项区

-  内容：用中文手写字体描述该API的使用易错点、兼容性要求、优化技巧，模板参考：

  1.  【易错点1】：如「避免【错误用法，如：重复绑定状态】，否则会导致【异常效果，如：弹窗重复弹出】」

  2.  【兼容性】：如「仅支持iOS15及以上系统，低版本系统需使用【兼容方案，如：UIKit桥接】」

  3.  【优化技巧】：如「【配置项】建议按需设置，过多冗余配置会影响界面渲染效率」

-  标注：用粉色荧光笔标注每条注意事项中的易错点、关键提示词。

  

## 三、排版要求（固定不变，适配手机竖屏阅读）

1.  竖屏9:16比例，内容从上到下按上述6个模块依次排列，模块之间用卡通分隔线（如虚线、小图标、彩色线条）区分，视觉层次清晰。

2.  手写字体大小分层：标题最大，模块副标题次之，正文内容适中，注释内容稍小，确保阅读舒适。

3.  代码框尺寸适中，占据屏幕宽度80%左右，代码字体清晰可辨，语法高亮明显。

4.  整体留白合理，无内容拥挤遮挡情况，符合手机竖屏保存、查阅、分享的使用场景。

  

## 四、使用说明

1.  直接替换所有「【目标SwiftUI API名称】」为你需要生成的API（如Alert、Text、Button、TextField等）即可使用。

2.  若API有特殊特性，可在对应模块中补充专属内容，不改变整体框架即可保持风格统一。
